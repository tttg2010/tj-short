# Seedance2 人脸合规经验

本页记录 TJ Short 在短剧带货图生视频里使用 Seedance2 / Seedance 2.0 时的可见人脸处理方法。

目标不是规避审核，而是把自有虚拟角色的来源和参考方式做清楚，同时保留短剧所需的人脸表演。

## 适用场景

- 短剧镜头需要演员表情，不能改成无脸镜头。
- 角色是原创虚拟演员或自有虚拟角色。
- 单张高拟真首帧被 Seedance2 判断为可能包含真人隐私信息。
- 需要继续输出真实摄影质感，而不是整片变成动画。

不适用：

- 未授权真人照片。
- 明星、网红、公众人物或相似脸。
- 任何试图绕过身份审核的真实人物素材。

## 经验结论

单张完整高拟真人脸首帧容易失败。只在 prompt 里写“原创虚拟演员”通常不够，因为审核会先看输入图本身。

如果平台或中转站支持“人物备案 / 真人备案 / 角色资产备案”，优先走备案资产链路：

```text
原创虚拟角色 -> 人物备案板 -> 平台备案 -> 备案资产 ID -> 用该备案资产生成 Seedance2 视频
```

从 AniShort 类工作流观察到的关键点：

- 备案状态更像后台资产记录，不是图片水印。
- 备案成功会返回平台内资产号。
- 备案资产通常绑定平台、项目、账号或素材 URL。
- 图片下载到本地后再重新上传，可能会失去备案状态。
- 服装、发型、年龄、脸部结构等强身份变化，可能需要重新备案。

公开文档里只写占位格式，例如 `asset-YYYYMMDDHHMMSS-xxxxx`，不要发布真实备案号、任务号、签名下载链接或私有上传路径。

不能伪造备案号，不能复用他人的备案资产，不能把 LibTV 等水印当成授权或备案证明。

## 官方提示词规则 + 生产判断

火山 Seedance2 文档有一个重要规则：即使请求体里使用 `asset://<asset ID>` 传入备案或素材库资产，提示词中仍然要用 `图片1`、`视频1`、`音频1` 这样的“素材类型 + 序号”来引用素材，不能直接写“asset-xxxx 是某某角色”。

生产上这样处理：

```text
请求体:
图片1 = asset://<filed_role_face_or_avatar_asset>
图片2 = 角色全身/半身服装参考
图片3 = 场景参考
图片4 = 产品参考

提示词:
将图片1中的面部特征、图片2中的服装造型定义为林夏。
图片3作为客厅场景参考，图片4只用于产品外形参考。
```

官方也提示人物多视图可能带来 ID 漂移或“双胞胎”问题。这里不要机械理解成“不能做人设板”。更准确的生产规则是：

- 人物备案板继续用于角色设计、备案、团队审核和资产管理。
- 真正送 Seedance2 生视频时，优先从备案板拆出单人素材：人脸特写 + 全身/半身服装图。
- 不要默认把完整多视图备案板作为唯一人物参考。
- 如果某个平台/模型实测完整备案板更稳，可以继续用，但要在 manifest 里记录模型、参数、结果和失败率。

硬门槛：Seedance2 可见人物项目必须先有 `角色主体库.md`、`人物备案板需求.md`、`Seedance2参考包计划.md`。没有这些文件时，不应继续生成首帧宫格、图生视频提示词或提交任务。

更稳定的路线是从首帧/宫格阶段就处理，而不是等视频提交失败后再补救：

- 正式首帧/宫格保持真实摄影质感。
- 只对人物脸部区域做 `face_pencil` 彩铅/手绘化。
- 身体、服装、产品、道具、场景、光影保持摄影质感。
- 同一镜头必须产出 3 张候选首帧：`face_pencil` 强、`face_pencil` 中、`blur_feature`（模糊主图 + 五官特写）。
- 逐张做 Seedance2 预检测；能过的候选里，选择“人脸信息保留最完整、表演最清楚”的那张。
- 失败图不重复硬撞，记录失败原因后换资产路线。
- `blur_feature` 是第三候选，不是等所有视频失败后才临时补救。

禁止把正式首帧整体降级成漫画、动画、插画、商业分镜风。整张漫画化只能作为风格草稿或预览图，不能作为 Seedance2 短剧带货的默认正式首帧。

产品无人物图、产品空镜或只拍道具的首帧，只适用于两类情况：产品证据镜头本身就是产品空镜，或 API 冒烟测试需要确认 salpx `/videos` 提交、轮询、下载是否可用。它不能作为“人物表演镜头被拒后”的默认替代路线；如果人物镜头三候选全部失败，必须先记录原因，再让用户确认是否重写镜头设计。

## 正式首帧三候选流程

每个 Seedance2 可见人物镜头，从宫格/首帧阶段就按同一镜头生成三候选：

1. `face_pencil_strong`：脸部处理更明显，身体、泳衣/服装、产品、动作、场景仍保持摄影质感。
2. `face_pencil_medium`：脸部处理适中，尽量保留表情、唇形、眼神和人物辨识信息。
3. `blur_feature`：模糊主图 + 五官特写图；主图控制构图、身体、服装、动作和场景，五官图补足原创虚拟角色脸部特征。

预检测规则：

1. 三候选逐张进 Seedance2 预检测或受控单镜头测试。
2. 每张记录 `candidate_id`、`method`、`precheck_status`、`reject_reason`、`face_info_score`、`acting_clarity_score`。
3. 在能通过的候选里，优先选择人脸信息、表演清晰度、唇形和眼神保留最完整的版本。
4. 不因为某张更“安全”就自动选最重处理；只要中等处理能过，通常优先中等处理。
5. 已失败候选不原图复投；只允许换处理强度、换参考包或换镜头设计。

## 默认方法：face_pencil

适合 Seedance2 可见人物首帧、宫格图和镜头参考。这个方法应前置到生产阶段，而不是等失败后才使用。

做法：

1. 保留原始首帧构图。
2. 只对人物脸部区域做彩铅、手绘或素描化。
3. 身体、服装、产品、宠物、道具、场景保持原始摄影质感。
4. 提示词中说明这是自有原创虚拟角色参考，并要求输出真实摄影质感。

关键点：

- 不要把整张图都变成彩铅。
- 不要把真人短剧首帧改成漫画角色。
- 不要裁掉脸。
- 不要改服装和身体动作。
- 脸部处理要明显，但边缘要自然。

## 备选方法：blur_feature 两张图

适合多人同框、高拟真脸、手机镜头、近脸镜头，或 `face_pencil` 失败后。

准备两张参考图：

- 图 A：主构图图。保留人物位置、服装、身体动作、道具、宠物、场景，只把脸部重度模糊。
- 图 B：五官特写图。把同一批虚拟角色的眼睛、鼻子、嘴巴等特征整理到白底图里。

提示词职责：

- 图 A 控制位置、服装、身体动作、场景和镜头构图。
- 图 B 控制虚拟角色五官特征。
- 输出恢复清晰自然完整人脸，并保持真实摄影质感。

接口经验：

- 如果接口的 `image` 字段只接受字符串，不要传数组。
- `image` 放模糊主图。
- 额外参考图放到模型支持的参考字段，例如 `images`。

## 推荐 Prompt

```text
参考图片 1 是本镜头主构图：保留人物站位、服装、身体动作、手机/道具位置、宠物位置、场景和镜头构图；图中人物脸部已做模糊，仅作为原创虚拟角色的构图参考。
参考图片 2 是同一批原创虚拟角色的五官特征整理图，用于补全各角色的眼睛、鼻子、嘴巴和脸部结构。
请输出真实摄影质感的短剧画面，恢复清晰自然的完整人脸，保持每个角色的脸型、发型、服装、动作、情绪和人物关系一致。该画面只包含原创虚拟演员，非真实人物、非公众人物、非明星脸、非未授权肖像。
画面内不要出现字幕、标题、对白气泡、水印或可读手机文字。
```

## 失败后判断

如果 `face_pencil` 失败：

- 不要重复提交同一张图。
- 按同一镜头三候选流程切换到另一张候选；如果强/中都失败，再使用 `blur_feature` 两张图。
- 不要把整张图改成漫画/动画风来规避。

如果 `blur_feature` 仍失败：

- 先补角色三视图或角色设定板。
- 降低单张参考图里的完整真人脸信息。
- 不要直接改成产品空镜或无脸镜头，除非用户明确接受牺牲人物表演并重写镜头设计。

## 生产原则

短剧带货的核心是冲突和表演。人脸镜头失败时，优先修复虚拟角色资产链路，而不是把镜头改成产品特写或无脸动作。
